# Shinobi Service

Сервис реестра шиноби деревни. Часть проекта [naruto-kage-crm](../README.md) — владеет
данными о шиноби и является единственным источником правды об их статусах.

## Ответственность

- Реестр шиноби: ранг, навыки, стихии, кеккей генкай, статус, статистика миссий.
- Жизненный цикл статусов: `available / reserved / onMission / injured / onLeave / disabled / dead / nukenin`. Смены статусов атомарны — двойное назначение на миссию невозможно даже при гонках.
- Резервирование шиноби под команды по запросу Dispatch service (gRPC), всё-или-ничего.
- Пополнение реестра только через выпуск из Академии: новый генин создаётся по событию из топика `student_events`, публичной операции «создать шиноби» нет.
- История смен статусов (NFR-4) — таблица history.

## API

### gRPC (внутренний, для Dispatch)

Контракт: [`api/shinobi/dispatch.proto`](api/shinobi/dispatch.proto).

- `GetAvailableShinobi` — список шиноби, доступных для назначения на миссию.
- `ReserveShinobi` — резерв батча шиноби под собираемую команду. Всё-или-ничего:
  при конфликте (кто-то уже занят) ничего не бронируется, в ответе возвращаются
  `conflicted_shinobi_ids` — Dispatch заменяет их и повторяет бронь.

### HTTP (админский) — планируется

Реестр и управление статусами для администрации деревни (отпуск, списание и т.п.).
